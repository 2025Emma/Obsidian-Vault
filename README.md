# Obsidian 基础仓库

这是一个用于快速搭建个人知识库的 Obsidian 基础仓库。它预先整理了常用目录、复盘模板、阅读模板、项目索引模板和一组高频插件配置，适合作为新知识库的起点，也适合作为个人系统持续迭代的底座。

这个仓库的重点不是把所有内容一次性填满，而是提供一个清晰、可复用、方便长期维护的知识管理框架。你可以从日记、项目、资源和归档四个入口开始记录，再通过日复盘、周复盘和月复盘把零散经验沉淀成可复用资产。

## 目录结构

```text
00 Inbox/
01 Projects/
02 Areas/
03 Resources/
04 Archives/
```

## 文件夹说明

| 文件夹 | 用途 |
| --- | --- |
| `00 Inbox` | 临时收集入口，用来放还没有分类的想法、素材、截图、灵感和待整理信息。 |
| `01 Projects` | 项目推进区，适合存放有明确目标、截止时间或交付结果的内容。 |
| `02 Areas` | 长期关注领域，用来管理健康、成长、关系、职业、财务等需要持续维护的主题。 |
| `03 Resources` | 资源库，存放模板、技能、参考资料、阅读笔记和可复用素材。 |
| `04 Archives` | 归档区，用来存放已经完成、暂停或不再活跃的项目和资料。 |

## 内置插件

仓库中保留了一组常用 Obsidian 插件，打开仓库后可以根据自己的需要启用或调整。

| 插件 | 作用 |
| --- | --- |
| Chinese Calendar | 支持农历、节日、节气和中国日历习惯，适合日记和周期复盘场景。 |
| Dataview | 用查询方式汇总笔记、任务、项目和元数据。 |
| Editing Toolbar | 提供类似文字处理软件的编辑工具栏，提高 Markdown 编辑效率。 |
| Image Toolkit | 增强图片预览、缩放、旋转和复制体验。 |
| Omnisearch | 提供更强的全文搜索能力。 |
| PDF++ | 增强 PDF 阅读、标注和 Obsidian 内联引用体验。 |
| Periodic Notes | 管理日记、周记、月记等周期性笔记。 |
| Claudian | 将 AI 协作工具嵌入 Obsidian 工作流，便于在库内进行文件读写、检索和多步骤协作。 |

## 如何下载这个仓库

仓库地址：

```text
https://github.com/2025Emma/Obsidian-Vault
```

### 方式一：直接下载 ZIP

适合不熟悉 Git，只想快速使用这套基础库的人。

1. 打开 GitHub 仓库页面：`https://github.com/2025Emma/Obsidian-Vault`。
2. 点击页面右上方的绿色按钮 `Code`。
3. 选择 `Download ZIP`。
4. 下载完成后解压压缩包。
5. 把解压后的文件夹移动到你想存放知识库的位置。
6. 打开 Obsidian，选择“打开本地仓库”，选中这个文件夹即可。

### 方式二：使用 Git 克隆

适合想持续同步更新、保留版本记录，或者后续自己也要用 GitHub 管理知识库的人。

```powershell
git clone https://github.com/2025Emma/Obsidian-Vault.git
```

克隆完成后，打开 Obsidian，选择“打开本地仓库”，再选择刚刚下载下来的 `Obsidian-Vault` 文件夹。

## 快速开始

1. 安装 Obsidian。
2. 通过 ZIP 下载或 Git 克隆，把这个仓库保存到本地。
3. 在 Obsidian 中选择“打开本地仓库”，并选择本仓库文件夹。
4. 根据提示启用社区插件。
5. 从 `00 Inbox` 开始收集临时内容。
6. 使用 `03 Resources/R-模板资源` 中的模板创建日记、周复盘、月复盘、阅读笔记或项目索引页。

## 推荐使用方式

### 1. 每日记录

每天使用日记模板记录任务、时间投入、关键产出和遗留待办。重点不是写得多，而是让每天的行动可以被回看、被汇总、被复盘。

### 2. 每周复盘

每周使用周复盘模板检查自己是否仍在主航道上，识别哪些事情真正产生了复利，哪些事情只是制造忙碌感。

### 3. 每月校准

每月使用月复盘模板检查目标完成情况、关键假设是否成立，以及下个月最值得投入的战略动作。

### 4. 项目管理

为每个重要项目创建一个项目索引页，把相关笔记、时间投入和阶段结论集中起来。这样项目不会散落在多个日记和临时笔记里。

### 5. 阅读转化

阅读模板的目标不是摘抄，而是把外部信息转化为自己的判断、方法和可复用资产。建议每次阅读都写清楚“我为什么现在读它”和“它能转化成什么行动”。

## 同步和版本管理

这个仓库适合配合 GitHub 做版本管理。建议在重要调整后提交一次版本，方便回滚和跨设备同步。

常用命令：

```powershell
git status
git add .
git commit -m "Update vault"
git push
```

如果在不同设备之间同步，请先拉取远端更新：

```powershell
git pull
```

## 注意事项

- 不建议把个人密钥、账号配置、临时会话记录和本地缓存上传到公开仓库。
- 如果你安装了新的插件，建议确认插件配置是否包含隐私信息后再提交。
- 空文件夹在 Git 中不会被保存，如需保留空目录，可以放置 `.gitkeep` 文件。
- 这个基础库是一个起点，不是固定规则。随着你的工作流变化，可以逐步调整目录、模板和插件。

## 维护建议

建议每隔一段时间检查一次：

- 哪些模板还在被持续使用。
- 哪些目录已经变成堆积区，需要重新整理。
- 哪些插件已经不再需要。
- 哪些复盘结论可以沉淀成长期方法论。

好的知识库不是一开始就完美，而是在持续使用中越来越贴合自己的真实工作和生活。
